import java.security.MessageDigest;

{% trans "บรรทัด 37 (Your custom variables):" %}

String sha256(String input) throws Exception {
    if (input == null) return null;
    MessageDigest md = MessageDigest.getInstance("SHA-256");
    byte[] hash = md.digest(input.getBytes("UTF-8"));
    StringBuilder sb = new StringBuilder();
    for (byte b : hash) {
        sb.append(String.format("%02x", b));
    }
    return sb.toString();
}

{% endif %} {% get_current_language as LANGUAGE_CODE %} {% if LANGUAGE_CODE == 'en' %}

The reason the Load Phase takes so long is because it uses the DB Writer Node, which processes one row at a time. The NodePit documentation, in the Batch size section of the DB Writer Node, states 'Entering 1 disables batch processing. This way only one row is processed at a time.' This means data is sent to the database one row at a time. When KNIME sends data row by row, MySQL needs to write data to disk every time a command is committed. According to the MySQL InnoDB Transaction Management documentation, 'Flushing and syncing log files at each transaction can limit the maximum number of transactions per second' and the speed of the storage device determines the maximum number of commands that can be executed per second.

{% elif LANGUAGE_CODE == 'zh-hans' %}

Load Phase 花费很长时间的原因是因为它使用 DB Writer Node，它一次处理一行。NodePit 文档在 DB Writer Node 的 Batch size 部分指出"输入 1 会禁用批处理。这样一次只处理一行。"这意味着数据一次发送到数据库一行。当 KNIME 逐行发送数据时，MySQL 需要在每次提交命令时将数据写入磁盘。根据 MySQL InnoDB Transaction Management 文档，"在每个事务刷新和同步日志文件可能会限制每秒的最大事务数"，存储设备的速度决定了每秒可以执行的最大命令数。

{% else %}

สาเหตุที่ Load Phase ใช้เวลานานเพราะว่ามีการใช้ DB Writer Node ซึ่งประมวลผลทีละ 1 แถว เอกสาร NodePit ระบุไว้ในหัวข้อ Batch size ของ DB Writer Node ว่า 'การใส่ค่า 1 จะปิดการประมวลผลแบบ Batch ซึ่งทำให้ประมวลผลทีละ 1 แถวเท่านั้น' หมายความว่าข้อมูลถูกส่งไปยังฐานข้อมูลทีละ 1 แถว เมื่อ KNIME ส่งข้อมูลทีละแถว MySQL จำเป็นต้องบันทึกข้อมูลลงดิสก์ทุกครั้งที่มีการยืนยันคำสั่ง ตามเอกสาร MySQL InnoDB Transaction Management ระบุว่า 'การบันทึกและซิงค์ไฟล์ log ในแต่ละธุรกรรมสามารถจำกัดจำนวนธุรกรรมสูงสุดต่อวินาทีได้' และความเร็วของอุปกรณ์จัดเก็บข้อมูลเป็นตัวกำหนดขีดจำกัดของจำนวนคำสั่งสูงสุดที่ทำได้ต่อวินาที

{% endif %}
